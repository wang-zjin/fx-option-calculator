# 测试团队工作说明

本文档说明测试团队的职责、问题提交流程，以及如何配合问题调试团队完成验收。

---

## 一、测试团队职责

1. **功能与回归测试**：对开发团队交付的 Web 界面与计算模块进行测试，覆盖：
   - 输入校验（日期顺序、数值范围、必填项）
   - 定价与 Greeks 计算（与需求书/参考案例一致）
   - 界面展示（输出项完整、单位正确、买卖方向符号正确）
   - 构建与运行（`npm run build`、`npm run dev`、`npm run preview`）

2. **问题提交**：发现问题后，按 [Issues/README.md](./Issues/README.md) 规范在 `Issues/` 下**新建 Issue**，使用 [Issues/_template.md](./Issues/_template.md) 填写：
   - **现象**：复现步骤、报错信息或错误结果
   - **原因**：简要技术原因（若已知）
   - **解决方案**：具体修改建议或步骤（便于调试团队执行）
   - **分类**：计算模块 / 前端与UI / 构建与部署 / 环境与工具 / 其他

3. **配合调试**：将 Issue 交给问题调试团队后，根据其修改进行**回归验证**，直到结果满意再关闭或更新该 Issue。

4. **用户说明**：负责向业务/使用方说明**如何使用与验证**（见 [如何使用与验证.md](./如何使用与验证.md)）。

---

## 二、问题提交流程（简要）

| 步骤 | 操作 |
|------|------|
| 1 | 根据问题类型选择 Issues 子文件夹（如 `Issues/前端与UI/`、`Issues/计算模块/`） |
| 2 | 新建文件，命名建议：`YYYY-MM-DD_简短描述.md`（如 `2025-01-30_校验缺少到期日.md`） |
| 3 | 按 `Issues/_template.md` 填写现象、原因、解决方案、相关链接 |
| 4 | （可选）在 `Issues/README.md` 末尾「问题索引」表添加一行，便于检索 |
| 5 | 将 Issue 路径或内容交给问题调试团队，待修复后回归验证 |
