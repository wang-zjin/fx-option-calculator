# 知识库 — 使用与调取指南

## 一、定位与目的

- **知识库**：集中存放项目相关的**参考文档、规格说明、规范与经验总结**，供各团队在执行工作时查阅与调用。
- **与 Issues 的区别**：Issues 记录「问题 + 解决方案」；知识库存放「可复用的知识、规格、规范」，作为日常工作的参考依据。
- **管理方式**：由知识库管理团队维护；新文件先放入 [待归纳](./待归纳/)，经归纳后放入对应分类文件夹。

---

| 知识库子文件夹 | 适用内容 | 主要使用团队 / 工作环节 |
|----------------|----------|--------------------------|
| [业务与需求](./业务与需求/) | 需求书、功能规格、范围确认、验收标准 | 需求与范围确认、产品/业务 |

| [构建与部署](./构建与部署/) | 构建配置、部署方式、环境变量、静态资源规范 | 联调与部署、运维 |

## 三、其他团队如何调取知识库

### 1. 按路径直接查阅

- 知道大致分类时，直接打开对应子文件夹（如 `知识库/定价与计算/`）查看或搜索文件。
- 各子文件夹内均有 README，说明该分类下应放什么、谁会用。

### 2. 在 Cursor / AI 中调用

- **按分类调用**（推荐）：  
  *「请读取 `知识库/定价与计算/` 下的文档，根据 [某规格] 实现/校验 [某功能]。」*
- **按文件调用**：  
  *「请结合 `知识库/业务与需求/功能规格-期权类型与定价.md` 检查当前实现是否满足输出规格。」*
- **全库检索**：  
  *「在 `知识库/` 下搜索与 [Greeks / 部署 / 输入校验] 相关的内容，并给出实现建议。」*

### 3. 检索关键词建议

| 若你需要… | 建议查阅的知识库分类 | 可搜关键词示例 |
|-----------|------------------------|----------------|
| 期权类型、输入输出、验收标准 | 业务与需求 | 功能规格、需求书、验收 |
| 定价公式、Greeks、数值方法 | 定价与计算 | Garman-Kohlhagen、二叉树、蒙特卡洛、Delta |
| 界面、交互、校验规则 | 前端与UI | 组件、输入校验、错误提示 |
| 打包、部署、环境变量 | 构建与部署 | Vite、Nginx、部署 |
| 开发环境、代码规范 | 环境与工具 | ESLint、Prettier、Node |

## 四、如何向知识库投递新文件

1. **投递**：将文件放入 **`知识库/待归纳/`**，无需事先分类。
2. **归纳**：知识库管理团队（或执行归纳的同事）会按「团队分工与需求」将文件移动到对应子文件夹，必要时更新本 README 或子文件夹说明。
3. **命名建议**：文件名尽量见名知意（如 `功能规格-期权类型与定价.md`、`Greeks-定义与单位.md`），便于后续检索。

---

## 五、与开发流程的关系

- 知识库支撑《开发流程设计》中的**需求与范围确认、技术选型、计算模块、Web UI、联调与部署**等环节。
- 执行某步骤时，可优先在对应分类下查找是否有现成规格或规范，再动手实现，保证与项目约定一致。
