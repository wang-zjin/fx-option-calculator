# Debug 机制说明 — Issues 知识库

**最近更新**：2026-01-30 · 共 5 条调试记录，索引已与当前 Issue 文件同步。

---

## 目的

- **记录**：开发/联调中遇到的问题及解决方案，便于复现与复用。
- **检索**：以后出现类似问题时，可让 AI 读取本文件夹中相似 Issue，参考已有处理方式。
- **迭代**：同一类问题有新解法或补充时，可更新对应 Issue 文件。

---

## 使用方式

### 1. 出现问题时的操作

1. **定位分类**：根据问题类型，选择对应子文件夹（见下方「分类说明」）。
2. **新建 Issue**：在该子文件夹下新建文件，建议命名：`YYYY-MM-DD_简短描述.md`（如 `2025-01-30_vite-build-白屏.md`）。
3. **填写内容**：按 [Issues/_template.md](./_template.md) 模板填写「现象 / 原因 / 解决方案 / 相关链接」。
4. **更新索引**（可选）：在本文档末尾的「问题索引」表中添加一行，方便后续检索。

### 2. 以后遇到类似问题时

- **对开发者**：在 Issues 对应分类下搜索关键词（或让 AI 在 `Issues/` 下搜索）。
- **对 AI**：可这样提问：
  - *「请读取 Issues 文件夹，找和 [当前现象/报错] 相似的问题，并参考已有解决方案。」*
  - *「根据 Issues 里关于 [某类问题] 的记录，给出处理步骤，如有新情况再更新对应 Issue。」*

### 3. 更新已有 Issue

- 若同一问题有更优解法、或补充了环境/版本信息，直接编辑对应 `.md` 文件即可。
- 若发现某 Issue 已过时（例如依赖升级后不再出现），可在该文件末尾注明「已过时」及原因。

## 分类说明

| 分类文件夹       | 适用问题类型 |
|------------------|--------------|
| [计算模块](./计算模块/) | 定价公式、Greeks、数值精度、单元测试、蒙特卡洛/二叉树等 |
| [前端与UI](./前端与UI/) | 界面渲染、交互、输入校验、错误提示、响应式布局等 |
| [构建与部署](./构建与部署/) | Vite/打包、部署（Nginx/Vercel）、环境变量、静态资源等 |
| [环境与工具](./环境与工具/) | Node/npm、ESLint/Prettier、依赖冲突、开发环境等 |
| [移动端与小程序](./移动端与小程序/) | iOS/Android 打包与运行、Capacitor/Xcode、键盘与触控、微信小程序等 |
| [其他](./其他/) | 无法归入以上类别的问题 |

如需新增分类，可在 `Issues/` 下新建子文件夹，并在此表中补充一行。
